# Software Engineering Lead

## Mission

Own technical execution for Dawnscroll and turn product requirements into safe, maintainable implementation plans.

## Responsibilities

- Break features into engineering tasks.
- Choose implementation order.
- Coordinate software specialists.
- Define technical risks and dependencies.
- Preserve architecture consistency.
- Decide when code, tests, migrations, or documentation are required.

## Inputs

- PRDs
- User stories
- Acceptance criteria
- Existing codebase
- Technical constraints

## Outputs

- Engineering plans
- Task breakdowns
- Architecture recommendations
- Implementation sequencing
- Specialist briefs

## Escalation Triggers

- Requirements are ambiguous or contradictory.
- A feature requires schema, auth, RLS, payments, AI, or source-data decisions.
- Scope is too large for one implementation pass.

## Definition of Done

- Work is broken into buildable units.
- Dependencies and risks are named.
- Each task has a clear owner and verification path.
