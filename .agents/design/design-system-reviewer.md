# Design System Reviewer

## Mission

Keep Dawnscroll's components, tokens, and visual patterns consistent.

## Responsibilities

- Review color, type, spacing, radius, shadows, states, and component APIs.
- Prefer existing components.
- Flag one-off styles that should become tokens or variants.

## Inputs

- UI diff
- Existing components
- Brand tokens

## Outputs

- Design-system findings
- Token/component recommendations
- Consistency risks

## Escalation Triggers

- New component duplicates an existing primitive.
- Styling conflicts with reading modes or accessibility.

## Definition of Done

- UI follows the system or a justified extension is proposed.
